This repo contains 35 production patterns for LLM systems — framework-agnostic reference implementations with benchmarks and cost analysis. It's organized by production concern (resilience, cost, observability, testing, safety, data pipeline, orchestration, performance), not by app type. Each pattern includes dual TypeScript + Python implementations, failure mode documentation, cost projections at three scales, and operational guidance. The repo is designed to be evergreen — it doesn't chase tools, it captures the patterns that keep showing up regardless of framework or model provider.

## Categories

### [Resilience](patterns/resilience/)
What happens when things go wrong. Fallbacks, retries, failover, circuit breaking. These patterns keep your system serving users when providers fail, latency spikes, or capacity drops.

### [Cost Control](patterns/cost-control/)
Making AI affordable at scale. Budgets, caching, routing, visibility. These patterns prevent runaway spend and help you understand where your money goes.

### [Observability](patterns/observability/)
Knowing what your system is doing. Tracing, quality monitoring, drift detection. These patterns make non-deterministic LLM behavior debuggable and monitorable.

### [Testing](patterns/testing/)
Verifying non-deterministic systems. Evaluation, regression, adversarial, snapshot. These patterns give you confidence that prompt changes and model updates don't break things.

### [Safety](patterns/safety/)
Guarding inputs and outputs. Validation, PII, injection defense, human oversight. These patterns protect users, data, and your system from harmful or unexpected behavior.

### [Data Pipeline](patterns/data-pipeline/)
The retrieval layer. Chunking, embeddings, index health, context management. These patterns ensure the data feeding your LLM is fresh, relevant, and properly structured.

### [Orchestration](patterns/orchestration/)
Keeping multi-step and multi-agent LLM systems alive. Loop guards, tool resilience, state recovery, routing. These patterns manage complexity when LLMs coordinate with tools and each other.

### [Performance](patterns/performance/)
Making LLM systems fast at scale. Latency budgets, batching, concurrency, backpressure. These patterns handle the reality that LLM calls are slow and rate-limited.
